# mkgrid-notes

## 项目简介

`mkgrid-notes` 是一个用于归档和发布独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，其目的是以结构化的方式存储和展示多个独立的 HTML 文件。每个页面可以是一个笔记、工具、演示或任何适合以静态 HTML 形式呈现的内容。

## 目录结构

```
mkgrid-notes/
├── index.html          # 仓库的入口页面，列出所有归档页面
├── pages/              # 存放所有独立 HTML 页面的目录
│   ├── page-01.html
│   ├── page-02.html
│   └── ...
├── assets/             # 共享资源目录（如 CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `index.html`：仓库的主页，用于导航到各个归档页面。
- `pages/`：每个独立的 HTML 文件存放于此，文件名建议使用有意义的名称，但不包含具体域名或关键词。
- `assets/`：存放所有页面共享的静态资源，如样式表、脚本和图片。

## 页面归档说明

- 每个 HTML 页面的内容应保持独立，避免依赖外部（非仓库内）资源。
- 页面可以包含任意适合静态展示的内容，如说明、工具、演示等。
- 文件名和内容中不应出现任何具体的域名、网址或营销性关键词。
- 所有页面应通过 `index.html` 进行索引，方便浏览。

## 维护说明

- 新增页面时，在 `pages/` 目录下创建新的 HTML 文件，并更新 `index.html` 中的导航链接。
- 如需添加共享资源，请放入 `assets/` 对应的子目录中。
- 请保持仓库的整洁：无用的页面或资源应及时移除。
- 本仓库为归档用途，不提供动态服务或后端功能。
